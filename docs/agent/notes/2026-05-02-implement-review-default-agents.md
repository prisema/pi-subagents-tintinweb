# Implement and Review default agents

## Goal
Add explicit Superpowers-style implementation and review subagents so the workflow has dedicated roles after `Explore` and `Plan`.

## Context
The default agent set had `general-purpose`, `Explore`, and `Plan`. `general-purpose` can write code, but it is a parent-twin catch-all, not a scoped TDD implementor. There was also no built-in evidence-driven reviewer role. We inspected Augment/Software Intent specialists under `/Users/rizzao/.augment/specialists/` and used the useful role split:
- `implementor.md`: scoped execution, no scope creep, verify before completion.
- `verifier.md`: acceptance criteria first, evidence before approval.
- `pr-reviewer.md`: high-confidence actionable review only.

## Decisions
- Added `Implement` as a default scoped TDD implementation agent.
- Added `Review` as a default read-only evidence-driven review/validation agent.
- Kept `general-purpose` as the broad parent-twin fallback.
- Kept `Explore` for context and `Plan` for Taskdone-ready planning artifacts.
- Gave both new roles FFF search extension access (`fffind`, `ffgrep`, `fff-multi-grep`).
- Updated the `Plan` Taskdone manifest contract so generated manifests encode Implement and Review behavior in `config.extraInstructions` and `config.qualityGate.instructions`.
